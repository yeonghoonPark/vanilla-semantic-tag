Semantic Tag란 무엇인가?

웹 접근성을 높이기 위한 의미를 가지는 tag

Semantic Tag로 얻을 수 있는 장점

  1. SEO
  2. Accessibility
  3. For developer, Maintainability

비슷한 Tag들의 의미와 차이점

  • <article> vs <section>

    article의 경우 blog post의 개별적post1개 또는 신문기사의 기사 1개를 분류할 때 사용한다.
    즉, article은 독립적으로 그 1개를 1페이지로 보여줘도 전혀 문제가 없을 경우에 사용할 수 있다.
    section의 경우 연관있는 내용들을 하나로 묶어 줄 때 사용한다.
    예를들어 article안에 많은 내용이 있을텐데 그중 연관있는 내용들을 하나로 묶어 주거나 분류할 때 사용한다.
    section이든 article이든 부모의 상하 관계는 정해져 있지 않다.
    위와 같은 기준을 따라 article안에 section이 있을 수 있고 section안에 article이 있을 수 있다.

  • <i> vs <em>

    i태그든 em태그든 둘 다 화면상으로는 동일하게 italic으로 보여지지만 screen-leader로 읽을 경우 아래와 같은 차이점을 가진다.
    i태그는 강조되지 않고 em태그는 강조되어 읽혀진다.
    따라서 style목적으로 사용한다면 i태그를 사용하고 web-accessibility를 고려한다면 em태그를 사용하면 된다.

  • <i> vs <em>

    i태그든 em태그든 둘 다 화면상으로는 동일하게 italic으로 보여지지만 screen-leader로 읽을 경우 아래와 같은 차이점을 가진다.
    i태그는 강조되지 않고 em태그는 강조되어 읽혀진다.
    따라서 강조 없이 style이 목적이라면 i태그를 사용하고 web-accessibility를 고려하여 강조하는 italic을 원한다면 em태그를 사용하면 된다.

  • <b> vs <strong>

    b태그든 strong태그든 둘 다 화면상으로는 동일하게 bold로 보여지지만 screen-leader로 읽을 경우 아래와 같은 차이점을 가진다.
    b태그는 강조되지 않고 strong태그는 강조되어 읽혀진다.
    따라서 강조 없이 style이 목적이라면 b태그를 사용하고 web-accessibility를 고려하여 강조하는 bold효과를 원한다면 strong태그를 사용하면 된다.

  • <ul> vs <ol> vs <dl>

    • ul의 경우 : 순서 없는 목록
    • ol의 경우 : 순서가 중요 할 경우
    • dl의 경우 : description list, 어느 한 단어에 대해 설명이 묶여 있을 경우, 자식태그로는 <dt> 와 <dd> 을 가진다.
      dt = description term, 설명 할 단어
      dd = description detail, 단어에 대한 설명

  • <img> vs CSS background-image

    img태그의 경우 해당 페이지에서 하나의 중요한 요소로 페이지와 연관되어 있다면 img태그를 사용한다.
    CSS background-image의 경우 해당 페이지와 연관 없이 style목적이라면 background-image를 사용한다.

  • <button> vs <a>

    button태그의 경우 client의 어떠한 action을 위해 필요한 경우에 사용한다.
    a태그의 경우 현재 페이지에서 다른 페이지로 이동하거나 그 페이지를 떠나는 경우에 사용한다.

  • <table> vs <CSS>

    table태그의 경우 정말 매우 많은 행과 열이 필요한 데이터를 넣을 경우에 사용한다.
    하지만 많은 행과 열이 필요없고 단순히 table처럼 보여지고 싶은 경우에는 CSS를 이용하여 다른 태그를 이용하길 권장한다.
